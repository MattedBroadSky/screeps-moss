# EnergyManager 详细设计 - 核心摘要

## 📋 基本信息
- **状态**: [草案] - 设计进行中
- **版本**: v1.0.0
- **优先级**: P0 (v1.0核心模块)

## 🎯 核心职责
1. **能量状态监控** - 实时监控采集、存储、消耗
2. **能量分配优化** - 智能分配到不同用途
3. **采集效率优化** - 优化Harvester分配
4. **供应链管理** - 全流程能量管理

## 🏗️ 关键设计决策

### 1. 三层管理架构
```
EnergyManager
    ├── 采集层 (Harvesting) - 能量源管理
    ├── 存储层 (Storage)    - 存储设施管理
    └── 分配层 (Allocation) - 需求分配管理
```

### 2. 智能分配策略
```javascript
分配优先级：
1. 紧急程度判断 → 关键/警告/正常
2. 用途优先级 → spawn > defense > upgrade > build > repair
3. 等待时间 → 等待时间长的优先
4. 历史表现 → 效率高的优先
```

### 3. 核心算法
- **Harvester分配算法** - 基于能量率和可采集点
- **优先级分配算法** - 多因素综合评估
- **能量预测算法** - 时间序列分析预测
- **紧急判断算法** - 动态调整紧急级别

## 📊 性能指标

### 目标指标
| 指标 | 目标值 | 说明 |
|------|--------|------|
| CPU使用/tick | < 3 | 能量管理专用CPU |
| 采集效率 | > 80% | 实际采集/理论最大 |
| 分配准确率 | > 90% | 需求预测准确率 |
| 中断恢复 | < 10 ticks | 能量中断恢复时间 |

### 监控重点
1. **能量利用率** - 整体能量使用效率
2. **采集平衡度** - 各能量源负载均衡
3. **存储分布** - 能量在存储设施中的分布
4. **需求满足率** - 各类需求的满足情况

## 🔗 依赖关系

### 输入依赖
1. **RoomState** - 房间状态数据
2. **Creep状态** - Harvester分配情况
3. **建筑状态** - 存储设施状态
4. **需求列表** - 各模块的能量需求

### 输出提供
1. **能量分配结果** - 给各模块分配能量
2. **采集优化建议** - Harvester分配建议
3. **能量状态报告** - 实时能量状态
4. **问题检测报告** - 能量相关问题

## 🧪 测试策略

### 测试场景
1. **正常场景** - 能量充足，正常分配
2. **紧缺场景** - 能量不足，优先级分配
3. **中断场景** - 能量源被破坏
4. **过载场景** - 需求超过供给
5. **恢复场景** - 从中断中恢复

### 验证要点
1. **分配公平性** - 各需求得到合理分配
2. **响应及时性** - 紧急需求快速响应
3. **预测准确性** - 能量预测接近实际
4. **恢复能力** - 中断后快速恢复

## 📈 扩展点

### 配置扩展
```javascript
configurable: {
  // 采集配置
  harvesting: { minPerSource: 2, maxPerSource: 4 },
  
  // 分配策略
  allocation: { 
    strategy: 'balanced', // balanced/emergency/growth
    priorities: { spawn: 10, defense: 9, upgrade: 7, build: 5, repair: 3 }

  // 紧急阈值
  emergency: { critical: 0.2, warning: 0.4, normal: 0.6 }
}
```
