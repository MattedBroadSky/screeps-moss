## 📋 基本信息

## 🎯 核心职责
1. **Creep生命周期管理** - 生成、续命、回收全周期
2. **身体配置优化** - 智能身体部件分配
3. **生成队列管理** - 优先级队列和调度
4. **性能监控** - Creep效率和健康监控
5. **内存管理** - 清理无效Creep内存

## 🏗️ 关键设计决策

### 1. 三层管理架构
```
CreepManager
    ├── 注册层 (Registry) - Creep跟踪和状态管理
    ├── 生成层 (Spawning) - 队列管理和身体优化
    └── 生命周期层 (Lifecycle) - 续命、回收、清理
```

### 2. 智能身体优化
```javascript
身体优化策略：
1. 能量充足 → 增强身体（增加工作/运输部件）
2. 能量适中 → 平衡配置（考虑角色需求）
3. 能量不足 → 简化身体（移除非关键部件）
4. 紧急情况 → 最小配置（保证基本功能）
```

### 3. 优先级队列系统
```javascript
优先级计算因素：
1. 角色重要性 (harvester > hauler > upgrader > builder)
2. 紧急程度 (critical > high > medium > low)
3. 等待时间 (等待越久优先级越高)
4. 能量效率 (成本低的优先)
5. 房间需求 (当前最需要的角色)
```

### 4. 生命周期管理
```javascript
生命周期决策：
1. 续命决策 → 价值评估 + 剩余时间 + 角色重要性
2. 回收决策 → 经济性分析 + 紧急需求 + 任务状态
3. 清理策略 → 定期清理 + 内存优化 + 性能监控
```

### 监控重点
1. **生成队列状态** - 队列长度、等待时间、成功率
2. **身体配置效率** - 成本效益比、角色适配度
3. **生命周期质量** - 平均寿命、续命率、回收率
4. **内存使用效率** - 内存清理效果、无效内存比例
